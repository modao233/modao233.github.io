在GitHub和Gitee上部署hexo的部分记录

一、创建托管仓库

  • GitHub，https://github.com/[username]
    仓库名格式为：[username].github.io
    创建仓库时添加一个Readme

  • Gitee，https://gitee.com/[username]
    仓库名完全和用户名相同：[username]
    创建时添加一个Readme

二、SSH公钥配置

我注册邮箱相同

若输出不为空，则复制其内容添加到托管服务的SSH Keys中

三、开启Pages服务

Github的Pages服务就先省略不说，这里专门说下Gitee的Pages服务，Gitee要求仓库里有个index.html才能开启Pages服务，在项目网页上手动添加项目文件index.html

Gitee每次部署后需要手动更新pages服务

如果Gitee上部署的hexo无法加载css和js，需要在hexo的配置文件里修改

https://gitee.com/[username]/[username]

  • url为用户空间 ：https://gitee.com/[username]
  • root为项目名称 ：/

访问页面为https://[username].gitee.io/
