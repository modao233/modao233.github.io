leetcode 733 图像渲染，涉及算法：DFS、BFS

题目描述

有一幅以二维整数数组表示的图画，每一个整数表示该图画的像素值大小，数值在 0 到 65535 之间。

给你一个坐标 (sr, sc) 表示图像渲染开始的像素值（行 ，列）和一个新的颜色值 newColor，让你重新上色这幅图像。

为了完成上色工作，从初始坐标开始，记录初始坐标的上下左右四个方向上像素值与初始坐标相同的相连像素点，接着再记录这四个方向上符合条件的像素点与他们对应四个方向上像素值与初始坐标相同的相连像素点，……，重复该过程。将所有有记录的像素点的颜色值改为新的颜色值。

最后返回经过上色渲染后的图像。

解题思路

解法一 DFS标记

思路

这道题感觉和前两天的“被围绕的区域”很像很像，也是对图的搜索标记相连区域，然后改变标记

  • 首先对图——一个二维矩阵进行DFS搜索，找到从给定位置出发所有和给的值相同的位置，标记为-1
    • 这里利用到给的条件里，值都大于0的性质
    • DFS的返回条件是行列范围跳出或值不相等或已被标记
  • 之后再对图进行遍历，找到所有标记为-1的位置，改成newColor完成渲染

复杂度分析

  • 时间复杂度：最坏情况下把每个位置走一遍，并且后来也是二重循环遍历完矩阵，所以时间复杂度为O(mn)
  • 空间复杂度：栈的深度最坏情况下为所有点都放进来，空间复杂度为O(mn)

解法二 BFS标记

思路

  • 使用队列来进行搜索，当搜索到一个满足条件的位置之后，会把他的上下左右入队，并且直接修改颜色防止重复入队
  • 当队空时，跳出循环，返回

复杂度分析

  • 时间复杂度：最坏情况下每个位置都入队，循环相当于遍历一个遍，所以是O(mn)
  • 空间复杂度：队列的开销最坏情况下也是O(mn)
